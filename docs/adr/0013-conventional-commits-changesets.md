# 0013 — Conventional Commits + Changesets

**Status:** Accepted
**Date:** 2026-06-11

## Context

Manual changelogs and release notes are error-prone and always out of date. Semver decisions are subjective without a systematic approach.

## Decision

- All commits follow [Conventional Commits](https://www.conventionalcommits.org/) spec, enforced by `commitlint` in CI and via husky pre-commit.
- Every PR that modifies a published package includes a changeset file (`.changeset/*.md`) generated by `pnpm changeset`.
- Release workflow: `pnpm changeset version` bumps package versions + generates CHANGELOG entries; `pnpm changeset publish` publishes to npm.

## Consequences

- CHANGELOG is auto-generated and always accurate.
- Semver bumps are driven by commit types (feat → minor, fix → patch, BREAKING CHANGE → major).
- Contributors must learn the commit format (documented in CONTRIBUTING.md).
