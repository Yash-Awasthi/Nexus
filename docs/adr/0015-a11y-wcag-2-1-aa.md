# 0015 — Accessibility Target: WCAG 2.1 AA

**Status:** Accepted (amended 2026-06-18 — Axe CI deferred to post-v1, lint rule kept)
**Date:** 2026-06-11

## Context

AI platforms are increasingly used in professional contexts where accessibility is a legal requirement (ADA, EN 301 549). Building in accessibility from the start is far cheaper than retrofitting. The `pnpm test:a11y` script exists in the root `package.json` but no `apps/ui` test:a11y script is wired — running it in CI would fail silently.

## Decision

**v1.0:** `eslint-plugin-jsx-a11y` is enabled in ESLint config (lint-time checks). New components must pass lint without a11y errors. No Axe CI scan gate yet.

**Post-v1.0:** Add Playwright + Axe integration test (`apps/ui/e2e/a11y.test.ts`) scanning all route pages. Wire into CI (`test:a11y` step). Require manual screen-reader pass before minor releases.

**Target remains:** WCAG 2.1 AA on all pages.

## Consequences

- v1 is protected at lint level — obvious a11y errors (missing alt, unlabelled buttons) are caught.
- Full automated Axe coverage is a post-v1 scope item.
- Manual screen-reader requirement deferred until Axe CI is passing.
- No blocking CI gate on Axe scans during active feature development.
