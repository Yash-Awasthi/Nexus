# 0015 — Accessibility Target: WCAG 2.1 AA

**Status:** Accepted
**Date:** 2026-06-11

## Context

AI platforms are increasingly used in professional contexts where accessibility is a legal requirement (ADA, EN 301 549). Building in accessibility from the start is far cheaper than retrofitting.

## Decision

- `pnpm test:a11y` runs Axe against every page route in CI.
- A manual screen-reader pass (VoiceOver or NVDA) is required before each minor release.
- Target: WCAG 2.1 AA compliance on all pages.
- `eslint-plugin-jsx-a11y` is enabled in ESLint config.

## Consequences

- Some UI components require additional ARIA attributes.
- CI is slightly slower due to Axe scans.
- Users with disabilities can use NEXUS without workarounds.
